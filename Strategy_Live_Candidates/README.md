# Strategy Live Candidates（實戰候選 / 部署卡）

本資料夾存放**回測基礎已全部通過、正在或即將進入實戰（demo / live）驗證**的策略。
每個策略一張「部署卡」，寫死驗證過的**最優設置**，作為實戰執行的單一真相來源。

## 研究 pipeline 分層

| 階段 | 資料夾 | 用途 |
|---|---|---|
| 1. 發想 | `Strategy_Ideas/` | 想法、設計草案、研究假說 |
| 2. 回測紀錄 | `Strategy_Records/` | 實際回測設定、結果、解讀、決策 |
| 3. **實戰候選** | **`Strategy_Live_Candidates/`** | **回測全關通過 → demo/live 驗證中的策略 + 部署卡** |

## 晉級門檻（進入本資料夾的條件）

一個策略要從 `Strategy_Records/` 晉級到此，至少需通過：

1. 全期 real ticks 正期望（PF、回撤可接受）。
2. 樣本外 / walk-forward（Forward 或滾動 OOS）未崩。
3. 穩定性（逐年 / 多窗多數為正，非單一 regime 撐起）。
4. **成本壓測**（含 commission，必要時含 swap/slippage）後 edge 維持。
5. **參數敏感度**：對每個關鍵參數單軸微調，確認落點是高原而非尖峰。門檻須**事前**寫定。
6. **參數傳遞驗證**：EA 與指標雙方都必須輸出參數，且逐項核對一致後，回測結果才可採信。

> ⚠️ 第 5、6 兩項為 2026-07-22 新增，起因是 `SR Channel Breakout` 的撤回。
> **第 1–4 關變動的都是「時間」，沒有一關變動「參數」**——刀鋒型的參數落點在整段歷史上都是同一個刀鋒，因此能輕鬆通過全部時間類檢驗。該策略的 OOS PF 達 1.654、逐年 6/6 為正，卻在鄰點崩壞至 0.84。
> 第 6 項則來自更早的一層失效：參數根本沒進到指標，而回測報告只印 EA 的請求值，看不出來。

未過以上者留在 `Strategy_Records/` 繼續研究或結案。

## 部署卡應包含

1. 狀態（demo forward / live）、幣種、週期、EA + 指標檔與版本
2. **完整 input 設置**（寫死、可直接套用）
3. 驗證摘要 + 連回 `Strategy_Records/` 的證據
4. 風控與部位規則
5. demo forward 檢查清單
6. **停用條件 (kill criteria)** —— 何時該下架/暫停

## 目前候選

**目前無候選。**

`SR Channel Breakout（EURUSD）` 曾於 2026-06-30 晉級，於 2026-07-22 撤回並隨策略結案移出本資料夾（部署卡與 `.set` 內容見 git 歷史）。撤回原因分兩層，兩者都值得後續晉級時引以為戒：

1. **晉級依據本身無效**：EA 以 positional `iCustom` 傳參，而 `input group` 會佔用一個參數位，導致指標實際收到的參數與部署卡記載的完全不同。回測報告只印 EA 的 input 值，不印指標實收值，因此錯位無法從報告中看出。
2. **還原真實參數後重驗仍失敗**：樣本外（OOS PF 1.654）與逐年（6/6 年正）都通過，但參數敏感度顯示該點是刀鋒——鄰點 PF 崩至 0.84–1.08，且 PF 與交易數呈完美單調反向，屬小樣本雜訊。

詳見 [Strategy_Records](../Strategy_Records/Strategy_SR_Channel_Breakout.md) §S10 與 §6.6。

## 注意

- 進入本資料夾**不等於 live ready**——demo forward 仍可能因實際滑點、swap、broker 差異或 live regime 而失效。
- 部署卡的設置一旦用於實戰，**任何更動都必須回到 `Strategy_Records/` 重新驗證**，不可在卡上直接調參數。
