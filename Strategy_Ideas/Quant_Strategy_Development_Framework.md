# 量化策略開發與想法架構（總綱）

建立日期：2026-07-04

狀態：方法論參考文件（不是策略想法；未來每個新想法都應該能在這份框架裡定位自己）

用途：發想新策略時，先回答「它屬於哪一類、吃哪種資訊、賭哪個機制、和手上既有的東西是否互補」，再決定要不要投入。避免重複開發同質策略、避免指標疊指標。

---

## 1. 策略分類：按「賺錢機制」分（最重要的分法）

策略的本質不是用了什麼指標，而是**賭市場的哪一種行為傾向**。同機制的策略高度相關，不同機制的策略才有組合價值。

| 機制 | 賭什麼 | 回報形狀 | 典型失效環境 | repo 案例 |
|---|---|---|---|---|
| **趨勢跟隨 / 動能** | 動過的會繼續動 | 低勝率、高 RR、正偏態 | 盤整年被反覆巴 | Turtle、SR Breakout、PrecisionSniper、Vegas Tunnel |
| **均值回歸** | 偏離會被拉回 | 高勝率、低 RR、負偏態 | 趨勢爆發日一次吐光 | 亞洲盤均值回歸、Weekend Gap Fade |
| **時段 / 季節性** | 特定時間有系統性訂單流 | 薄而穩的漂移 | 市場結構改變（alpha decay） | FX 時段效應、黃金日內季節性、London Breakout |
| **事件驅動** | 資訊發布後的定價過程 | 事件級肥尾 | 日曆資料源、滑價 | （無；工程量大，暫不開發） |
| **套利 / 相對價值** | 相關商品價差收斂 | 極高勝率、爆倉尾部 | 相關性 regime 崩解 | （無；零售 FX 幾乎做不到） |
| **資金管理型（偽機制）** | 靠加倉攤平「製造」勝率 | 曲線漂亮、尾部藏爆倉 | 一次大趨勢 | Martingale / Grid 系列（**不進研究管線**） |

**規則一：新想法必須先說出自己的機制。** 說不出「賭什麼行為傾向、為什麼這個傾向存在」的想法，只是指標排列組合。

**規則二：組合價值 = 機制多樣性。** 兩個趨勢策略換不同指標仍是同一注；一個趨勢 + 一個時段策略才是兩注。

## 2. 策略分類：按「資訊來源」分（判斷正交性用）

| 資訊來源 | 內容 | 特性 |
|---|---|---|
| **價格衍生（指標）** | MA、RSI、MACD、ATR、Donchian、SuperTrend、QQE… | 全部同源（都是價格的函數）；兩個價格指標相疊 ≠ 兩份資訊，只是減少交易數 |
| **時間結構** | session、開收盤、星期、fix 時點 | 與價格指標天然正交；參數面積小 |
| **市場結構** | S/R、pivot、區間、跳空 | 半正交（仍來自價格，但編碼的是位置不是動能） |
| **量 / 流動性** | tick volume、spread 行為 | FX 只有 tick volume（代理品質有限），但仍是獨立維度 |
| **跨市場** | 相關商品、DXY、利差 | 正交性高；工程與資料需求高 |
| **事件 / 基本面** | 經濟日曆、央行 | 正交性最高；MT5 回測支援最差 |

**規則三：一個策略最多一個「主資訊源」+ 一到兩個「正交過濾」。** Vegas Tunnel 檔第 8 節的教訓：Tunnel（價格）+ QQE（價格）= 同源疊加，主要效果只是降頻。若當時改成 Tunnel（價格）+ session（時間），才是真正的兩維。

## 3. 策略解剖：每個策略都是同一條流水線

```text
[Regime 過濾] → [Setup 條件] → [Trigger 觸發] → [進場執行] → [出場] → [風控] → [組合層]
```

| 層 | 職責 | 可用元件 | 原則 |
|---|---|---|---|
| **Regime 過濾** | 現在該不該玩這個策略 | 慢速 MA 方向、ATR 水位、session、星期 | 一層就好；它砍交易數最兇 |
| **Setup** | 環境是否就位 | 區間形成、價格靠近 S/R、波動收縮 | 定義「什麼情況下訊號才有意義」 |
| **Trigger** | 此刻進場的扳機 | 突破收盤、指標翻轉、定時 | 只能有一個；兩個 trigger = 兩個策略 |
| **進場執行** | 怎麼進 | 收盤確認 vs stop order、點差檢查 | 已收盤棒（`shift=1`）是 repo 鐵律 |
| **出場** | 怎麼離開 | ATR SL / RR TP / 時間出場 / 追蹤 | **出場決定回報形狀**，常比進場重要 |
| **風控** | 單筆與整體損失上限 | 風險% 手數、MaxHolding、Friday exit | 沿用 repo 既有慣例，不重造 |
| **組合層** | 多策略怎麼共存 | magic number 隔離、相關性檢查 | 見第 6 節 |

**規則四：每一層只放一個元件，加任何一層都要記錄交易數變化。** 交易數掉一半而 OOS 沒改善 = 那層是過擬合。

## 4. 指標與過濾器工具箱（按功能分類，不按名氣）

### 4.1 指標按功能分

| 功能 | 指標 | 適合放哪層 |
|---|---|---|
| 趨勢方向 | MA 系（EMA/SMA）、SuperTrend、Donchian 中軸 | Regime |
| 動能強度 | RSI、MACD、QQE、Force Index | Trigger |
| 波動水位 | ATR、BB 寬度、區間高度 | Setup 過濾 / SL 定距 |
| 位置結構 | S/R channel、pivot、Donchian 邊緣 | Setup / 位置過濾 |
| 量能 | tick volume vs 均量 | 突破確認過濾 |

同功能的指標互為替代品，不能相疊（RSI+MACD+QQE 三重確認 = 一份動能資訊算三遍 + 三倍參數面積）。

### 4.2 過濾器類型與代價

| 過濾器 | 擋什麼 | 代價 |
|---|---|---|
| 時段（session） | 流動性差 / 機制不在的時段 | 低（時間參數不易過擬合） |
| 點差（MaxSpread） | 成本異常時刻 | 近零（純防禦，必開） |
| 波動（ATR 區間） | 太死或太瘋的市況 | 中（上下限是兩個可調參數） |
| Regime（慢 MA） | 逆勢訊號 | 高（砍交易數最兇、趨勢定義本身可調） |
| 位置（S/R 距離） | 撞牆進場 | 高（依賴上游指標品質；見 SNR filter 研究） |
| 量（VolMult） | 假突破 | 中（FX tick volume 代理品質有限） |
| 事件日 | 已知的分布外日子 | 低但要日曆資料 |

**規則五：過濾器加壓順序 = 代價從低到高。** MaxSpread → session → 波動 → 量 → regime → 位置。每加一個都要過「OOS 有沒有真的改善」關。

## 5. 想法評估計分卡（新想法寫檔前先過這關）

每個新想法檔的第 8 節「誠實評估」實際上在回答這六項：

| 維度 | 問題 | 好答案長什麼樣 |
|---|---|---|
| **機制可信度** | 為什麼這個 edge 存在、誰在另一邊送錢 | 有經濟/行為解釋（訂單流、制度、成本結構），不是「線圖上看起來會這樣」 |
| **證據等級** | 誰驗證過 | 期刊論文 > 含成本的獨立回測 > 部落格回測 > 從業者敘事 > TradingView 讚數 |
| **參數面積** | 有幾個可調參數 | ≤ 3 個可調 = 好；每多一個，OOS 崩掉的先驗翻倍 |
| **成本敏感度** | 單筆毛利是全成本的幾倍 | ≥ 4× = 安全；< 2× = 大概率死在成本，先手算再寫程式 |
| **組合正交性** | 和手上已有的東西相關嗎 | 不同機制 + 不同資訊源 + 不同持倉時窗 |
| **證偽成本** | 最便宜的否定方法是什麼 | 能用手算/純統計先證偽的最好（如 Weekend Gap 的統計關） |

**規則六：先找最便宜的證偽路徑。** 順序永遠是：手算損益兩平 → 純統計掃描 → 固定參數 baseline → 才輪到優化。任何在 baseline 為負的想法直接結案，不進優化。

## 6. 策略組合原則（多策略共存）

- **回報形狀互補**：正偏（趨勢）配負偏（均值回歸）配穩定漂移（時段），整體曲線比任一單策略平滑。
- **時間錯開**：London Breakout（08:00–17:00 London）、亞洲盤 MR（03:00–07:00）、時段效應（固定窗口）天然不重疊 → 保證金與風險不堆疊。
- **相關性驗收**：新策略晉級前，把它的回測日損益與既有組合的日損益算相關；ρ > 0.5 就要問「它真的是新的一注嗎」。
- **隔離執行**：每策略獨立 magic number；Netting 帳戶注意 SR Breakout EA 的 ownership 防護行為（他人部位存在時拒進場）。
- **同引擎多假說**：session 區間引擎（Breakout/Fade 兩 mode）、定時進出引擎（FX 時段 + 黃金季節性）——**引擎數 < 策略數**是工程上的正確方向。

## 7. repo 現況在此框架中的定位圖

```text
機制             既有/研究中                     backlog（想法檔）
─────────────────────────────────────────────────────────────────
趨勢/動能        PrecisionSniper (USDJPY M15)    Vegas Tunnel + QQE
                 SR Breakout (EURUSD H1) ★晉級   London Breakout
                 Turtle（無紀錄，待補 baseline）
均值回歸         （無）                          亞洲盤 MR、Weekend Gap Fade
時段/季節性      （無）                          FX 時段效應、黃金日內季節性
事件驅動         （無）                          （暫不開發）
偽機制           Martingale / Grid（除名候選）
```

看得出來：**既有資產全部集中在「趨勢/動能」一格**——這就是為什麼 2026-07 的五個新想法全部押在時段與均值回歸兩格，補的是機制多樣性，不是更多趨勢變體。

## 8. 從想法到實盤的關卡（對接既有 workflow）

```text
想法檔（本資料夾，含第 5 節計分卡自評）
  → 最便宜證偽（手算 / 統計掃描）        ← 不過 = 結案，成本一天內
  → 固定參數 baseline（real ticks 長樣本） ← 負期望 = 結案，不優化
  → 年度拆分 + OOS/Forward + 成本壓測      ← 標準見 Strategy_Records 第 8 節
  → 小幅 A/B（每次只動一層）              ← IS 改善但 OOS 沒有 = data mining
  → Strategy_Records 建檔 → 部署卡 → demo forward → 小額實盤
```

倒在越早的關卡越好——**研究產能的主要浪費是在低先驗想法上跑了太多輪優化**（SR Channel 通用線的教訓），不是想法不夠多。

## 9. 想法來源清單（未來去哪裡挖）

按證據等級排：

1. **學術異象文獻**（SSRN、期刊）：時段效應、日曆效應、announcement drift——注意發表後 alpha decay，一律要 2020+ 子樣本驗證。
2. **含成本的獨立回測站**：QuantifiedStrategies、QuantRocket blog、Quantpedia——規則透明的才值得跟。
3. **既有策略的機制轉移**：同一機制換商品/換時窗（SR Breakout 的 EURUSD 專用化就是這條路成功的例子）。
4. **既有策略的鏡像**：突破策略的反面是 fade 策略（London Breakout ↔ 亞洲盤 MR），引擎免費。
5. **TradingView / 零售圈熱門模板**：證據等級最低，只當「機制提示」用，指標本身照第 4 節拆解重組，不照抄。
6. **自家回測的虧損分析**：既有策略在哪種日子虧最多？那種日子可能是另一個策略的主場。
